**1) 위젯 꾹 눌러 편집 — 운동 선택 UI**

소형 위젯을 `StaticConfiguration` → `AppIntentConfiguration` 으로 전환해서, 홈 화면에서 위젯을 꾹 누르면 iOS 기본 "위젯 편집" 시트가 뜨고 거기서 푸시업/풀업/스쿼트 중 하나 고르게 됨.

핵심 코드 3개:

```swift
// ExerciseChoice — 위젯 편집 시트에 보여줄 선택지
enum ExerciseChoice: String, AppEnum {
    case pushup, pullup, squat
    static var caseDisplayRepresentations: [ExerciseChoice: DisplayRepresentation] = [
        .pushup: DisplayRepresentation(title: "푸시업"),
        .pullup: DisplayRepresentation(title: "풀업"),
        .squat:  DisplayRepresentation(title: "스쿼트")
    ]
}

// 위젯 설정 intent
struct NudgeSingleConfigIntent: WidgetConfigurationIntent {
    @Parameter(title: "운동", default: .pushup) var exercise: ExerciseChoice
}

// Provider — AppIntentTimelineProvider 로 바뀜
struct NudgeSingleProvider: AppIntentTimelineProvider { ... }
```

위젯도 두 개로 분리했다:
- `NudgeSingleWidget` (kind: "NudgeSingleWidget") — `AppIntentConfiguration`, `systemSmall`, 편집으로 운동 선택
- `NudgeTriWidget` (kind: "NudgeTriWidget") — `StaticConfiguration`, `systemMedium`, 3개 병렬 (설정 없음)

Bundle 에 둘 다 등록.

**2) 앱 아이콘 A안 적용 (Ripple Tap)**

`icon-concepts/A-ripple-tap.svg` (teal 그라디언트 배경 + 흰색 이중 링 + 중앙 점) 을 기반으로 iOS 18 싱글-1024 레이아웃의 3종 PNG 생성:

- `AppIcon-light.png` — 원본 teal 그라디언트
- `AppIcon-dark.png` — 짙은 teal 배경 + 라이트 민트 링
- `AppIcon-tinted.png` — 검정 배경 + 흰색 링 (tinted 모드용)

전부 1024×1024 RGB(알파 제거 — Apple 스펙). `Assets.xcassets/AppIcon.appiconset/Contents.json` 의 light/dark/tinted 3개 appearance 에 각각 `filename` 항목 추가.

변환 파이프라인: cairosvg → Pillow flatten → PNG. 모두 세션 안에서 해결.

**3) Phase 4 통계 화면**

일단 데이터 저장소 확장 없이 기존 `[YYYY-MM-DD: [exercise: count]]` JSON 그대로 쓰는 "라이트" 옵션으로 감.

`SharedStore` 추가:
```swift
static func counts(on date: Date) -> [Exercise: Int]
static func recentDays(_ days: Int) -> [(date: Date, counts: [Exercise: Int])]
static func recordedDateRange() -> (oldest: Date, newest: Date)?
```

`StatsView.swift` 신규:
- 상단 세그먼트: 주간(7일) / 월간(4주) / 연간(12개월)
- 기간 합계 카드 (큰 숫자 + numericText contentTransition)
- 스택 바 차트 (Swift Charts `BarMark`, 운동별 색 `chartForegroundStyleScale` 매핑)
- 최근 12주 히트맵 (주간 탭 한정, 월요일 시작, 5단계 민트 그레이스케일)
- 운동별 합계 카드 (원형 dot + 횟수)
- 빈 상태("기록이 없어요")

`RootView.swift` 신규 — TabView 로 `오늘(ContentView)` / `통계(NavigationStack + StatsView)` 분리. `nudgeApp.swift` 엔트리도 `ContentView()` → `RootView()` 로 변경.

## 🤔 결정 사항

- **통계 저장소: 가벼운 확장 채택** — 탭 타임스탬프 저장이 아니라 날짜별 카운트만 유지. Nudge 페르소나("가볍게 +1 만") 에 "18시~21시에 주로 운동합니다" 같은 세분 인사이트는 과한 분석. 맥스아웃이 필요로 하면 그쪽으로 보내는 게 맞음.
- **위젯 중형은 Static 유지** — 중형은 3종 다 보여주는 게 가치인데 편집으로 "중형에서도 1종만 보기" 옵션을 주는 건 선택지만 늘리고 가치가 없음.
- **아이콘은 A안 단일 적용** — B/C/D 는 preview.html 에 남겨두되 실제 에셋은 A안만. Tinted 변형은 배경을 검정으로 깔아 시스템 틴트가 흰색 링 위에 자연스럽게 얹히도록 함.
- **통계 탭에 NavigationStack, 오늘 탭에는 NavigationStack 없음** — 통계는 navigationTitle 로 "통계" 표시가 자연스럽고, 오늘 탭은 이미 자체 header 가 있어 이중 네비바 피하려고 뺌.

## 🐛 문제와 해결 과정

**문제 1 — tinted 아이콘 톤 잡기**
- iOS 18 tinted 는 "회색 톤 단일 레이어" 컨벤션인데, 링이 흰색/투명으로 겹치면 시스템이 잘 인식 못할 수 있음
- 해결: 검정 배경 + 흰색 링의 높은 대비 버전 사용. 시스템이 흰색 영역을 틴트 컬러로 정상 덮어줌.

**문제 2 — Provider 타입 호환**
- 소형은 `AppIntentTimelineProvider`, 중형은 `TimelineProvider` 로 다른 프로토콜 → 같은 `Timeline<NudgeEntry>` 로 합쳐서 쓸 수 있는지 애매
- 해결: Provider 둘로 나눔. 엔트리 타입 `NudgeEntry` 는 공유. 각 Provider 가 각자의 Widget 하나만 담당.

Swift Charts `chartForegroundStyleScale` 로 운동별 색 도메인 매핑하는 거 처음 써봤는데 문자열 키로 짝짓는 방식이 직관적이고, `Exercise.accentColor` 를 그대로 재사용할 수 있어서 앱 컬러 시스템이 한 곳에서 유지됨.
